# Contributing to GAL CLI

Thank you for your interest in contributing to GAL CLI!

## How to Contribute

### Reporting Issues

- Use GitHub Issues for bug reports and feature requests
- Check existing issues before creating new ones
- Provide clear reproduction steps for bugs

### Development

GAL CLI is developed in the private [Scheduler-Systems/gal](https://github.com/Scheduler-Systems/gal) monorepo.

If you're interested in contributing code:
1. Contact us at team@schedulersystems.com
2. We'll discuss the contribution and provide access if appropriate

### Documentation

Documentation improvements are always welcome! Submit issues with suggestions or corrections.

## Code of Conduct

We are committed to providing a welcoming and inclusive experience for everyone.

### Our Standards

- Be respectful and constructive
- Focus on what is best for the community
- Show empathy towards other community members

### Enforcement

Unacceptable behavior may be reported to team@schedulersystems.com.

## Questions?

- Email: team@schedulersystems.com
- Issues: [GitHub Issues](https://github.com/Scheduler-Systems/gal-cli/issues)
